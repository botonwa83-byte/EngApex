# EngApex · 英语登顶 开发方案

> 参考 app：ChemApex（工程目录 `../chemapex`），本项目是 Apex 家族的英语版，
> 最大化复用其架构与设计语言（4-Tab / 主线 / SM-2 复习 / 错题本 / 付费墙 / DesignSystem），
> 但灵魂模块按英语学科特性"换心脏"——**心脏不是静态视觉资产，而是一台活的算法引擎**。

## 一、产品定位

**一句话**：它不是又一个背单词 / 刷题 app——它是一台高考英语的"提分导航仪"，
永远算得出你下一个 +5 分在哪，用最短路径带你拿到。

- **主战场**：高考英语（高中冲刺），资源占 ~90%。
- **目标用户**：分数在 90–135 区间、想再冲一个台阶的学生（提分空间最大、ROI 最高）。
- **天花板**：135+ 的读后续写高分句式 / 难句长难句拆解，"向上仰望"的付费钩子。

**核心判断**：英语不是化学换皮。高考英语的灵魂是**可计算的得分结构**——
150 分能精确拆成 7 个模块，每个题型都有有限的考点和固定的解题决策树。
这就是算法的用武之地：把"老师的提分直觉"算法化。

## 二、灵魂：三个算法引擎（本项目最大差异点）

ChemApex 的灵魂是"元素星图"这类静态杀手级视觉资产；
EngApex 的灵魂是三个端上纯逻辑引擎，构成一条闭环：

### ① 估分器 `EstimatorEngine`
按新高考结构把 150 分拆开：

| 模块 | 分值 | 题量 |
|------|------|------|
| 听力 | 30 | 20×1.5 |
| 阅读理解（四篇） | 37.5 | 15×2.5 |
| 七选五 | 12.5 | 5×2.5 |
| 完形填空 | 15 | 15×1 |
| 语法填空 | 15 | 10×1.5 |
| 应用文写作 | 15 | — |
| 读后续写 | 25 | — |

根据做题正确率 × 难度加权，**实时估算高考分**，每模块一根分数条 + 置信度。
让"提分"第一次变成看得见的数字。

### ② 提分雷达 `GainRadar`（最大创新点）
对每个考点 / 模块计算**边际提分 ROI**：

```
ROI = (失分空间 × 可提升概率 × 高考权重) ÷ 预计投入时间
```

- 失分空间 = 满分 − 当前估分
- 可提升概率 = 该考点近期错误率 × 可学性系数（规则型高、玄学型低）
- 高考权重 = 模块分值占比
- 永远把用户导向"投入产出比最高的那 +5 分"

**学习路径不按教材章节排，按提分性价比动态排序**——这是 app 真正的卖点。

### ③ 错因诊断树 `DiagnosisEngine`
每道错题沿决策树定位根因 → 四类标签：

```
词汇盲区 / 语法漏洞 / 题型策略错 / 粗心审题
```

归因结果自动回喂给 GainRadar（更新可提升概率）和 SM-2 复习（生成复习卡），形成闭环。

## 三、脊梁：主线「提分之路」（提分七关）

主线 = 7 个得分模块，按"中段冲高分"学生的典型 ROI 排序（规则强、易提分的在前）。
GainRadar 叠加在主线之上，动态推荐"下一关该打哪"。

| # | 关卡 | 模块 | 核心算法/得分点模型 |
|---|------|------|---------------------|
| 1 | 语法的骨架 | 语法填空(15) | 有提示词/无提示词二分决策树（词形 vs 连词冠词介词） |
| 2 | 词块与逻辑 | 完形填空(15) | 逻辑词 / 固定搭配 / 语境复现 三类考点决策树 |
| 3 | 衔接的信号 | 七选五(12.5) | 代词/连接词/复现词 衔接信号打分 |
| 4 | 阅读的定位 | 阅读理解(37.5) | 细节/主旨/推理/词义猜测 四题型定位策略 |
| 5 | 模板与亮点 | 应用文(15) | 体裁模板骨架 + 亮点句注入 |
| 6 | 情节与续写 | 读后续写(25) | 情节链算法 + 按情节阶段推荐高分句式 |
| 7 | 听力的预判 | 听力(30) | 场景词预激活 + 设问预判（二期） |

**节点结构**：学（考点卡/决策树）→ 练（题型靶场）→ 测（估分校准，刷新雷达）。
**解锁规则**：完成本关诊断校准即点亮——但雷达可随时建议"插队"打高 ROI 关。

## 四、题型 = 得分点模型 + 解题决策树（吃透考点）

每个题型不是简单"题+答案"，而是结构化的：`考点标签 → 决策树步骤 → 陷阱 → 同类迁移`。
- **完形**：每空标注三类考点之一，给"前后看→定词性→定逻辑→排除"四步。
- **阅读**：每题标题型，配定位策略（题干关键词回文定位 / 同义替换识别 / 推理边界）。
- **七选五**：每空高亮衔接信号词，算法演示打分过程。
- **语法填空**：先判有无提示词，再走对应分支（时态/语态/非谓语 vs 连词/介词/冠词）。
- **读后续写**：情节六要素链（起因→发展→高潮→转折→结局→升华），每段配高分句式库。
- **应用文**：体裁模板 + 三档亮点句（高级词/从句/倒装强调）。

## 六、诚信红线

- 不臆造"某年某卷某题"出处（沿用 Apex 家族红线）。
- **英语特有**：
  - 所有例句、搭配、语法规则必须准确（学生会直接背进考场）。
  - **不做假的"AI 批改"**：读后续写/应用文提供高分范文 + 句式库 + 自评 rubric，
    由学生对照评分，不假装机器给出可信分数。
  - 估分器明确标注为"参考区间 + 置信度"，不承诺等于真实高考分。

## 七、技术方案

- SwiftUI / iOS 16 / Swift 5 / XcodeGen，bundle id `com.engapex.app`。
- **纯端上算法 + 内置题库，离线、零 API**——与系列一致、买断制、不烧外部模型。
- DesignSystem / ThemeColors / SM-2 复习 / 错题本 / Streak / PurchaseManager / Paywall
  整套复用 ChemApex（机械改名，不重写）。
- 三个引擎为新增纯逻辑：`EstimatorEngine` / `GainRadar` / `DiagnosisEngine`，
  代码量小、可单元测试，是本项目核心资产。
- 英语文本天然适合纯文本渲染，无需 KaTeX / 特殊渲染。

## 八、复用与省 token 策略

- **骨架靠 shell 复制**：`cp -R ../chemapex` → 改名 `ChemApex→EngApex`、`com.chemapex→com.engapex`，
  几乎零模型 token 拿到 ~8k 行基础设施。模型只动灵魂层。
- **内容与代码分离**：题库/句式库/词块库 schema 先定死，分批灌数据（可后台批量产）。
- **算法先于内容**：三引擎写一次即稳，避免边写边返工。

## 九、分期路线

- **一期 MVP（本次）**：4-Tab + 提分驾驶舱（估分器+雷达）+ 提分之路 6 关（语法填空/完形/
  七选五/阅读/应用文/读后续写）+ 题型靶场（每型 ~15 题，带决策树解析）+ 错因诊断 +
  句式/词块库 ~30 条 + 错题本。`xcodebuild` 可构建。
- **二期**：听力关 + GainRadar 完整 ROI 调参 + SM-2 复习完整版 + 题库扩充 + 熟词僻义档案。
- **三期**：读后续写情节链互动训练 + 付费墙（对标 ¥22 买断）+ 难句拆解 / 135+ 进阶内容。
